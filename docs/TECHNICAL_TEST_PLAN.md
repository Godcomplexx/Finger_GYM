# Программа и методика технических испытаний

## Объект испытаний

Программный модуль Finger GYM версии `0.2.0`, включающий трекинг кисти, расчет метрик, оценку упражнений, формирование рекомендации, журналирование и сохранение JSON-протокола.

## Цель

Подтвердить функциональную корректность, устойчивость к ошибкам входных данных, воспроизводимость результатов, полноту протокола и готовность к интеграции с основной системой АПК.

## Испытательный стенд

- ПК с Windows;
- Python 3.10+;
- веб-камера 720p/30 fps или выше;
- MediaPipe Hand Landmarker;
- набор тестовых видео/синтетических кадров;
- pytest для автоматизированной проверки.

## Тест-кейсы

| ID | Требование | Метод | Критерий приемки |
|---|---|---|---|
| V-01 | Запуск модуля | Запуск `python main.py` | Камера и модель инициализируются |
| V-02 | Отсутствие камеры | Запуск без доступной камеры | Рекомендация не формируется, ошибка фиксируется |
| V-03 | Калибровка | Сессия с валидной рукой | Калибровка получает `is_ready=true` |
| V-04 | Непригодная калибровка | Нет руки в кадре | Протокол сохраняется без рекомендации |
| V-05 | Потеря трекинга | Кадры с `is_valid=false` | При `validTrackingRatio < 0.65` результат `unreliable` |
| V-06 | Корректность упражнений | Автотесты по синтетическим точкам | Все unit-тесты проходят |
| V-07 | Категория качества | Баллы 75/50/49 и плохой VTR | Категории `good/medium/poor/unreliable` |
| V-08 | Версии и модель | Проверка JSON | Есть версия модуля, алгоритма и SHA-256 модели |
| V-09 | Журнал событий | Типовая сессия | В JSON есть обязательные события |
| V-10 | Интеграционный формат | Проверка JSON schema | Все обязательные поля передаются |
| V-11 | ПДн | Аудит протокола | Нет прямых ФИО, используется код пациента |
| V-12 | Воспроизводимость | Повторный прогон набора | Расхождение итогового балла в пределах допуска |

## Текущая автоматическая проверка

```bash
python -m pytest
```

## Нерешенные испытания

- испытания на реальных видеоданных;
- сравнение с экспертной оценкой;
- стресс-тесты камер и освещения;
- проверка интеграции с основной системой АПК;
- аудит ИБ и доступа к данным.
