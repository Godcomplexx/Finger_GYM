# Документация для раздела 3.X

Этот каталог содержит рабочий комплект документов для приведения модуля Finger GYM к требованиям раздела 3.X технического задания и дальнейшей подготовки доказательной базы.

Документы являются проектными заготовками. Они не заменяют юридическую оценку регистрационного маршрута, клиническую валидацию и утверждение порогов профильным врачом/методистом.

## Состав

- `INTENDED_USE.md` - назначение, границы применения и ограничения.
- `SECTION_3X.md` - готовый текст раздела 3.X для ТЗ.
- `TRACEABILITY_MATRIX.md` - матрица соответствия нормативным требованиям.
- `INTERFACE_SPEC.md` - формат JSON-протокола и обмена с основной системой АПК.
- `DATA_PROTECTION.md` - требования к ПДн, обезличиванию, ролям и журналам.
- `DATASET_VALIDATION_PLAN.md` - план датасетов, разметки и аналитической/клинической валидации.
- `VIDEO_DATA_REQUIREMENTS_AND_DATASETS.md` - требования к видеоданным, условиям съемки, клинической интерпретации и внешним датасетам.
- `TECHNICAL_TEST_PLAN.md` - программа и методика технических испытаний.
- `RISK_REGISTER.md` - начальный реестр рисков и мер управления.
- `ULTRALEAP_COMPARISON_PLAN.md` - план разработки версии на Ultraleap и сравнения с MediaPipe.

## Статус нормативных ссылок

Статусы нормативных документов должны перепроверяться перед выпуском ТЗ, ПМИ и регистрационного пакета. Текущая редакция этих документов подготовлена для разработки и внутренней проверки, а не для подачи регистрационного досье без дополнительной экспертизы.

## Экспорт в Word

Word-версии документов формируются в папку `docs/docx/` командой:

```bash
python scripts/export_markdown_to_docx.py
```

Используемый профиль оформления: A4, поля 30/15/20/20 мм, Times New Roman, основной текст 14 пт, межстрочный интервал 1.5, абзацный отступ 1.25 см, выравнивание по ширине, табличные границы и нумерация листов в нижнем колонтитуле.
